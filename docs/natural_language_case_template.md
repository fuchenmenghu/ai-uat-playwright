## 编写要求

- 页面定位信息必须清楚，优先使用输入框 label、按钮文本、表格列名、行匹配规则、菜单路径。
- 如果存在更稳定的技术定位方式，可以补充 CSS selector、XPath、data-testid、id 或 name。
- 不要只写“点击查询”“查看状态”，必须说明定位哪个元素、输入什么值、匹配哪一行。
- 通过“行匹配规则”定位到目标行后，如果需要核对行内某列的值，在“预期结果”中用自然语言说明字段名、期望值来源和比较方式。
- 如果缺少页面元素定位信息，脚本生成前必须补充，不允许猜测。
- 需要跨步骤传递的单据号必须显式写明变量名。
- 如果步骤需要提取页面变量，在“需要提取的页面字段”和“变量名”中一一对应填写；通过“行匹配规则”定位到行后，可用自然语言描述要提取的行内字段。
- 如果“需要提取的页面字段”或“变量名”任一为空，则该步骤不执行变量提取处理。
- `涉及系统` 用于执行前打开登录页签，多个系统用英文逗号分隔，例如：`cibo,wms`。

## 推荐示例

Step 01

- 系统：WMS系统
- 模块/页面：采购单
- 菜单路径：采购 → 采购单
- 操作描述：进入采购单页面，使用 `source_order_no` 查询采购单。
- 页面定位信息：
  - NCC单号输入框：label = NCC单号
  - 查询按钮：button text = 查询
  - 结果表格：采购单列表
  - 行匹配规则：NCC单号列 = `{source_order_no}`
- 预期结果：查询结果中存在 NCC单号 等于 `{source_order_no}` 的记录。
- 断言/等待规则：
  - 使用轮询判断
  - 最多查询 3 次
  - 每次间隔 10 秒
  - 若 3 次后仍未查询到记录，则判定为不通过
- 提取变量：
  - 需要提取的页面字段：采购单号
  - 变量名：`wms_purchase_order_no`
